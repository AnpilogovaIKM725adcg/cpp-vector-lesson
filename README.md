# cpp-vector-lesson
# 📦 Навчально-методичний комплекс: Шаблони класів та динамічний масив на C++

Цей документ містить розгорнутий план презентації (послайдово) та готовий до використання квіз (тест) для перевірки знань студентів на основі кастомної реалізації класу `Vector`.

---

## 📑 ЧАСТИНА 1: Структура та вміст презентації

### Слайд 1: Титульний
* **Заголовок:** Реалізація власного динамічного масиву (Vector) на C++
* **Підзаголовок:** Використання шаблонів класів (Templates) та ручне керування пам'яттю
* **Мета лекції:** Розібрати внутрішній устрій динамічних контейнерів та навчитися працювати з шаблонами.

---

### Слайд 2: Що таке шаблон класу (Class Template)?
* **Основна ідея:** Механізм C++, який дозволяє створювати класи-трафарети, здатні працювати з будь-якими типами даних без дублювання коду.
* **Синтаксис:** 
  ```cpp
  template <class T>
  class Vector { ... };
  ```
* **Перевага:** Один і той самий код класу `Vector` підійде для створення `Vector<int>`, `Vector<double>` або `Vector<string>`.

---

### Слайд 3: Анатомія класу Vector (Поля private)
Динамічний масив керується трьома основними атрибутами:
1. `T* data` — вказівник на перший елемент масиву у динамічній пам'яті (купі / heap).
2. `size_t length` — фактична кількість елементів, які користувач уже додав.
3. `size_t capacity` — загальний розмір виділеного буфера пам'яті (місткість).

---

### Слайд 4: Життєвий цикл: Конструктор та Деструктор
* **Конструктор по замовчуванню:**
  * Задає початковий розмір `length = 0`.
  * Виділяє базову пам'ять під 2 елементи: `capacity = 2`.
  * Викликає оператор `new T[capacity]`.
* **Деструктор:**
  * Обов'язково звільняє ресурси.
  * Використовує `delete[] data` (квадратні дужки важливі, бо видаляється масив об'єктів).

---

### Слайд 5: Механізм автоматичного розширення (метод resize)
Коли масив заповнюється повністю (`length >= capacity`), виконуються такі кроки:
1. Нова місткість подвоюється: `capacity = capacity * 2`.
2. Виділяється новий тимчасовий блок пам'яті: `T* temp = new T[capacity]`.
3. Усі наявні елементи копіюються зі старого масиву в новий через цикл `for`.
4. Стара пам'ять звільняється через `delete[] data`.
5. Вказівник `data` перенаправляється на новий масив `temp`.

---

### Слайд 6: Основні методи для роботи з даними
* `push_back(T item)` — додає елемент в кінець. Якщо місця немає — спочатку викликає `resize()`.
* `pop_back()` — логічно видаляє останній елемент. Фізично пам'ять не зменшується, просто зменшується лічильник `length`.
* `at(size_t index)` — повертає посилання (`T&`) на елемент за індексом. Дозволяє як читати, так і змінювати значення.
* `size()` — константний метод, що повертає поточну кількість елементів.

---

### Слайд 7: Практичний запуск та демонстрація
Аналіз функції `main()`:
1. Створюється об'єкт `Vector<int> numbers;` (пам'ять виділено під 2 елементи).
2. Додаються `10` та `20` (масив заповнений).
3. Додається `30` -> спрацьовує `resize()`, місткість стає `4`.
4. Виведення на екран покаже: `10, 20, 30`.
5. Після `pop_back()` та повторного виведення на екрані залишаться лише `10, 20`.

### Питання 2

- [ ] А) Пам'ять під останній елемент одразу звільняється через оператор delete.
- [x] Б) Зменшується лише значення змінної length, фізично розмір буфера пам'яті не змінюється.

- [ ] Г) Відбувається повне очищення масиву та деструкція всіх елементів.

### Питання 3
За якої умови всередині методу `push_back()` викликається приватний метод `resize()`?
- [ ] А) Коли масив повністю порожній (`length == 0`).
- [x] Б) Коли поточна кількість елементів досягла або перевищила місткість (`length >= capacity`).
- [ ] В) Після кожного третього виклику методу додавання елемента.

### Питання 4

- [x] В) `delete[] data;` — оскільки пам'ять виділялася під динамічний масив за допомогою `new T[]`.
- [ ] Г) Пам'ять звільняти не потрібно, вона очиститься автоматично при виході з зони видимості.

## 🔑 ЧАСТИНА 3: Ключ до тесту з детальними поясненнями для викладача

1. **Правильна відповідь: В**
   * *Обґрунтування:* Рядок `capacity = 2;` у тілі конструктора жорстко задає стартовий розмір буфера.
2. **Правильна відповідь: Б**
   * *Обґрунтування:* Реалізація методу містить лише `length--;`. Зменшення лічильника робить елемент "невидимим" для циклів, але фізично чарунка пам'яті в купі залишається зайнятою до деструктора.
3. **Правильна відповідь: Б**
   * *Обґрунтування:* Код перевірки: `if (length >= capacity) { resize(); }`. Це класичний алгоритм динамічного масиву.
4. **Правильна відповідь: В**
   * *Обґрунтування:* Невідповідність операторів виділення (`new[]`) та видалення (`delete` замість `delete[]`) призводить до невизначеної поведінки програми (Undefined Behavior) та витоку пам'яті.
5. **Правильна відповідь: В**
   * *Обґрунтування:* Початкова ємність 2. Додавання 10 та 20 заповнюють її. При додаванні 30 спрацьовує формула `capacity = capacity * 2` (разом 4). Метод `pop_back()` зменшує лише кількість елементів (`length` стає 2), але ємність назад не стискається.
6. **Правильна відповідь: В**
   * *Обґрунтування:* Повернення посилання дозволяє методу працювати в лівій частині оператора присвоювання (як l-value).
