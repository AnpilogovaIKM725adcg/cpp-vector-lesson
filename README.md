# cpp-vector-lesson
# 📦 Шаблони класів та динамічний масив на C++ (Кастомний Vector)

Цей репозиторій містить навчальні матеріали для вивчення внутрішнього устрою динамічних контейнерів у C++ на прикладі шаблону класу `Vector`.

---

## 📑 ЧАСТИНА 1: Структура презентації (План лекції)

<details>
<summary>📋 Розгорнути план презентації (Слайди 1-7)</summary>

### Слайд 1: Титульний
* **Заголовок:** Реалізація власного динамічного масиву (Vector) на C++
* **Підзаголовок:** Шаблони класів (Templates) та ручне керування пам'яттю.

### Слайд 2: Що таке шаблон класу (Class Template)?
* **Ідея:** Створення класів-трафаретів для роботи з будь-якими типами даних без дублювання коду.
* **Синтаксис:** `template <class T> class Vector { ... };`
* **Перевага:** Один код підходить для `Vector<int>`, `Vector<double>` тощо.

### Слайд 3: Анатомія класу Vector (Поля private)
Динамічний масив керується трьома атрибутами:
1. `T* data` — вказівник на масив у динамічній пам'яті (купі / heap).
2. `size_t length` — фактична кількість доданих елементів.
3. `size_t capacity` — загальний розмір виділеного буфера (місткість).

### Слайд 4: Життєвий цикл: Конструктор та Деструктор
* **Конструктор:** Задає `length = 0`, виділяє базову пам'ять під 2 елементи (`capacity = 2`) через `new T[capacity]`.
* **Деструктор:** Обов'язково звільняє ресурси через `delete[] data` для запобігання витокам пам'яті.

### Слайд 5: Механізм автоматичного розширення (метод resize)
Коли масив заповнюється повністю (`length >= capacity`):
1. Місткість подвоюється: `capacity = capacity * 2`.
2. Виділяється новий тимчасовий блок пам'яті: `T* temp = new T[capacity]`.
3. Елементи копіюються зі старого масиву в новий через цикл.
4. Стара пам'ять звільняється через `delete[] data`.
5. Вказівник `data` перенаправляється на `temp`.

### Слайд 6: Основні методи для роботи з даними
* `push_back(T item)` — додає елемент в кінець. Якщо місця немає — викликає `resize()`.
* `pop_back()` — логічно видаляє елемент (зменшує лічильник `length`, пам'ять не стискається).
* `at(size_t index)` — повертає посилання (`T&`) на елемент для читання та модифікації.
* `size()` — повертає поточну кількість елементів.

### Слайд 7: Практичний запуск та демонстрація
Аналіз функції `main()`:
1. Створюється `Vector<int> numbers;` (capacity = 2).
2. Додаються `10` та `20` (масив заповнений).
3. Додається `30` -> спрацьовує `resize()`, місткість стає `4`.
4. Виведення на екран покаже: `10, 20, 30`.
5. Після `pop_back()` на екрані залишаться лише `10, 20`.
</details>

## 🧠 ЧАСТИНА 2: Інтерактивний квіз для самоперевірки
*Ви можете натискати мишкою на чек-бокси `[ ]`, щоб обрати варіант, а потім відкривати спойлер для перевірки.*

### ❓ Питання 1
Яке початкове значення місткості (capacity) встановлюється в конструкторі класу `Vector`?
- [ ] А) 0
- [ ] Б) 1
- [ ] В) 2
- [ ] Г) 10

<details>
<summary>🔍 Перевірити правильну відповідь на Питання 1</summary>

**Правильна відповідь:** **В) 2**  
*Обґрунтування:* У тілі конструктора класу чітко прописано рядок: `capacity = 2;`. Саме такий початковий розмір буфера виділяється в пам'яті.
</details>

### ❓ Питання 2
Що відбувається з виділеною пам'яттю, коли викликається метод `pop_back()`?
- [ ] А) Пам'ять під останній елемент одразу звільняється через оператор delete.
- [ ] Б) Зменшується лише значення змінної length, фізично розмір буфера пам'яті не змінюється.
- [ ] В) Місткість масиву (capacity) автоматично зменшується вдвічі.
- [ ] Г) Відбувається повне очищення масиву.

<details>
<summary>🔍 Перевірити правильну відповідь на Питання 2</summary>

**Правильна відповідь:** **Б) Зменшується лише значення змінної length, фізично розмір буфера пам'яті не змінюється**  
*Обґрунтування:* Реалізація методу містить лише зменшення лічильника: `length--;`. Елемент стає недоступним для циклів, але сама пам'ять у купі залишається зарезервованою.
</details>

### ❓ Питання 3
За якої умови всередині методу `push_back()` викликається приватний метод `resize()`?
- [ ] А) Коли масив повністю порожній (`length == 0`).
- [ ] Б) Коли поточна кількість елементів досягла або перевищила місткість (`length >= capacity`).
- [ ] В) Після кожного третього виклику методу додавання елемента.
- [ ] Г) Коли індекс елемента стає більшим за максимальне значення `size_t`.

<details>
<summary>🔍 Перевірити правильну відповідь на Питання 3</summary>

**Правильна відповідь:** **Б) Коли поточна кількість елементів досягла або перевищила місткість (length >= capacity)**  
*Обґрунтування:* Перед додаванням нового елемента код перевіряє `if (length >= capacity)`. Якщо вільних комірок у виділеному буфері не залишилось, викликається розширення масиву.
</details>

### ❓ Питання 4
Який оператор керування пам'яттю використовується в деструкторі класу `Vector` і чому?
- [ ] А) `delete data;` — тому що data є звичайним вказівником.
- [ ] Б) `free(data);` — оскільки це стандартна функція мови С.
- [ ] В) `delete[] data;` — оскільки пам'ять виділялася під динамічний масив за допомогою `new T[]`.
- [ ] Г) Пам'ять звільняти не потрібно, вона очиститься автоматично.

<details>
<summary>🔍 Перевірити правильну відповідь на Питання 4</summary>

**Правильна відповідь:** **В) delete[] data;**  
*Обґрунтування:* Оскільки масив створювався через оператор виділення пам'яті для масивів `new T[capacity]`, парний деструктор обов'язково повинен мати квадратні дужки `delete[]`. Інакше виникне помилка витоку пам'яті.
</details>

### ❓ Питання 5
Якою буде місткість (`capacity`) об'єкта `numbers` наприкінці виконання функції `main()` з наданого коду?
- [ ] А) 2
- [ ] Б) 3
- [ ] В) 4
- [ ] Г) 8

<details>
<summary>🔍 Перевірити правильну відповідь на Питання 5</summary>

**Правильна відповідь:** **В) 4**  
*Обґрунтування:* Стартова ємність 2. При додаванні 3-го елемента (числа 30) умова `length >= capacity` (2 >= 2) стає істинною і викликається `resize()`. Формула `capacity = capacity * 2` збільшує ємність до 4. Наступний виклик `pop_back()` зменшує лише `length`, тому ємність залишається рівною 4.
</details>

### ❓ Питання 6
Що дозволяє робити повернення типу `T&` (посилання на тип) у методі `at(size_t index)`?
- [ ] А) Тільки швидко зчитувати елемент без копіювання.
- [ ] Б) Автоматично перевіряти, чи не виходить індекс за межі масиву.
- [ ] В) Використовувати метод як для читання, так і для модифікації елемента (наприклад, `numbers.at(0) = 5;`).
- [ ] Г) Видаляти елемент за вказаним індексом.

<details>
<summary>🔍 Перевірити правильну відповідь на Питання 6</summary>

**Правильна відповідь:** **В) Використовувати метод як для читання, так і для модифікації елемента**  
*Обґрунтування:* Повернення значення за посиланням (`T&`) дозволяє методу працювати як l-value (знаходитися ліворуч від знака присвоювання), що дає змогу перезаписувати дані в масиві.
</details>
