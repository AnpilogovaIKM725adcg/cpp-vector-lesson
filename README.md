### ❓ Питання 1
Яке початкове значення місткості (capacity) встановлюється в конструкторі класу `Vector`?
- [ ] А) 0
- [ ] Б) 1
- [ ] В) 2
- [ ] Г) 10

<details>
<summary>🔍 Натисніть тут, щоб перевірити відповідь на Питання 1</summary>

**Правильна відповідь:** **В) 2**  
*Пояснення:* У тілі конструктора класу чітко прописано рядок: `capacity = 2;`. Саме такий початковий розмір буфера виділяється в пам'яті під майбутні елементи.
</details>

### ❓ Питання 2
Що відбувається з виділеною пам'яттю, коли викликається метод `pop_back()`?
- [ ] А) Пам'ять під останній елемент одразу звільняється через оператор delete.
- [ ] Б) Зменшується лише значення змінної length, фізично розмір буфера пам'яті не змінюється.
- [ ] В) Місткість масиву (capacity) автоматично зменшується вдвічі.
- [ ] Г) Відбувається повне очищення масиву.

<details>
<summary>🔍 Натисніть тут, щоб перевірити відповідь на Питання 2</summary>

**Правильна відповідь:** **Б) Зменшується лише значення змінної length, фізично розмір буфера пам'яті не змінюється**  
*Пояснення:* Реалізація методу містить лише зменшення лічильника: `length--;`. Елемент стає недоступним для стандартних циклів програми, але сама пам'ять у купі залишається зарезервованою.
</details>

### ❓ Питання 3
За якої умови всередині методу `push_back()` викликається приватний метод `resize()`?
- [ ] А) Коли масив повністю порожній (`length == 0`).
- [ ] Б) Коли поточна кількість елементів досягла або перевищила місткість (`length >= capacity`).
- [ ] В) Після кожного третього виклику методу додавання елемента.
- [ ] Г) Коли індекс елемента стає більшим за максимальне значення `size_t`.

<details>
<summary>🔍 Натисніть тут, щоб перевірити відповідь на Питання 3</summary>

**Правильна відповідь:** **Б) Коли поточна кількість елементів досягла або перевищила місткість (length >= capacity)**  
*Пояснення:* Перед додаванням нового елемента код робить перевірку `if (length >= capacity)`. Якщо вільних комірок у виділеному буфері не залишилось, запускається процес перевиділення пам'яті та копіювання.
</details>

### ❓ Питання 4
Який оператор керування пам'яттю використовується в деструкторі класу `Vector` і чому?
- [ ] А) `delete data;` — тому що data є звичайним вказівником.
- [ ] Б) `free(data);` — оскільки це стандартна функція мови С.
- [ ] В) `delete[] data;` — оскільки пам'ять виділялася під динамічний масив за допомогою `new T[]`.
- [ ] Г) Пам'ять звільняти не потрібно, вона очиститься автоматично.

<details>
<summary>🔍 Натисніть тут, щоб перевірити відповідь на Питання 4</summary>

**Правильна відповідь:** **В) delete[] data;**  
*Пояснення:* Оскільки масив створювався через оператор виділення пам'яті для масивів `new T[capacity]`, парний оператор деструкції обов'язково повинен мати квадратні дужки `delete[]`. Якщо їх забути, виникне витік пам'яті або краш програми.
</details>

### ❓ Питання 5
Якою буде місткість (`capacity`) об'єкта `numbers` наприкінці виконання функції `main()` з наданого коду?
- [ ] А) 2
- [ ] Б) 3
- [ ] В) 4
- [ ] Г) 8

<details>
<summary>🔍 Натисніть тут, щоб перевірити відповідь на Питання 5</summary>

**Правильна відповідь:** **В) 4**  
*Пояснення:* Стартова ємність дорівнює 2. При додаванні 3-го елемента (числа 30) умова `length >= capacity` (2 >= 2) стає істинною і викликається `resize()`. Формула `capacity = capacity * 2` подвоює ємність до 4. Наступний виклик `pop_back()` зменшує лише `length` (кількість елементів стає 2), але ємність назад не стискається і залишається рівною 4.
</details>

### ❓ Питання 6
Що дозволяє робити повернення типу `T&` (посилання на тип) у методі `at(size_t index)`?
- [ ] А) Тільки швидко зчитувати елемент без копіювання.
- [ ] Б) Автоматично перевіряти, чи не виходить індекс за межі масиву.
- [ ] В) Використовувати метод як для читання, так і для модифікації елемента (наприклад, `numbers.at(0) = 5;`).
- [ ] Г) Видаляти елемент за вказаним індексом.

<details>
<summary>🔍 Натисніть тут, щоб перевірити відповідь на Питання 6</summary>

**Правильна відповідь:** **В) Використовувати метод як для читання, так і для модифікації елемента**  
*Пояснення:* Повернення значення за посиланням (`T&`) дозволяє методу працювати як l-value (тобто перебувати ліворуч від знака присвоювання `+=` або `=`), що дає змогу перезаписувати дані всередині масиву.
</details>
